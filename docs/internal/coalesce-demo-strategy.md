## The Thesis

**Agents need their own support layer.** Human support tools (Plain, Pylon, Zendesk) are built for conversations between humans. Agent support is a fundamentally different shape of problem — transactional, synchronous, structured, 7-second resolution. You can't bolt agent support onto a human support platform any more than you can bolt real-time payments onto a check-clearing system.

## Audiences

**Afore VCs (cold email + video):** Need to believe the B2A support category exists and Coalesce is first. They'll challenge: "Is this a feature or a company?" and "Why won't Plain just add this?"

**AgentMail cofounder:** Needs to see why AgentMail should adopt Coalesce as its agent support layer instead of routing agents to email/Plain. He already suggested the email approach — the demo needs to break that belief.

## The Structural Argument (for investors)

Plain and Pylon cannot build this without building a new company inside their company:

| Dimension | Human Support (Plain/Pylon) | Agent Support (Coalesce) |
|-----------|---------------------------|-------------------------|
| Format | Natural language conversation | Structured JSON with fix_steps[] |
| Latency | Minutes to hours | 7 seconds |
| State | Threaded, persistent | Transactional, stateless |
| Auth | User accounts | Signed URLs (zero friction) |
| Knowledge | Support history + KB | API docs + OpenAPI specs |
| Resolution | Human reads and acts | Agent parses and retries |
| Buyer | Head of Support | Platform Engineer |

## Challenge Prep (Investor Q&A)

**"This is a feature, not a company."**
Building a support endpoint is easy. Handling millions of agent resolution requests with economies of scale, multi-tenant knowledge management, and cross-API behavioral intelligence is infrastructure. I scale infra for a living — that's literally my current job.

**"Where's the demand?"**
AgentMail processes 8M requests daily. Browser Use is facing the same problem — their agents fail on third-party APIs with no resolution path. Every agent-serving company hits this.

**"The 4% organic discovery rate is low."**
For every agent that finds Coalesce, 100 silently fail and the human doesn't know until morning. The problem isn't the 4% that find us — it's the 96% of agent errors that currently go completely unresolved. Coalesce makes them visible and resolvable.

**"Why now?"**
Claude Code, Cursor, MCP, and the first wave of B2A companies made agents a real API consumer segment in 2025-2026. Before that, agent traffic was negligible. Now AgentMail alone handles millions of agent API calls daily.

**"Why won't Plain just add this?"**
Plain sells to support teams. We sell to platform engineers. Different buyer, different architecture, different response format. Plain would need to build a new synchronous API endpoint, a new AI pipeline for structured JSON output, a new auth system for signed URLs, and sell to a buyer they've never talked to. That's not a feature — that's a new company inside their company.

**"Why won't API companies just build this themselves?"**
They could — and some will try. But each company building their own resolution endpoint means duplicating the same AI pipeline, the same docs ingestion, the same prompt engineering, the same multi-turn conversation logic. That's like every SaaS company building their own payment system instead of using Stripe. When Coalesce resolves an auth error for AgentMail, that resolution pattern improves diagnosis for every other API on the platform. That cross-API intelligence is the network effect that no single company can replicate in-house.

**"What's the team?"**
Two technical cofounders already in production. Tanishq (infra/backend, currently at AgentMail — scales systems handling 8M requests/day) and Abhinit (full-stack, building the resolution engine and integrations). Both active, both technical, both building.

**"What's the revenue model?"**
Per-resolution pricing. Free tier (1,000 resolutions/month) to drive adoption, then usage-based. Platform fee for the dashboard. At scale: $100-3K/month per customer depending on traffic.

**"What do you do with $250K?"**
Partner with 5-10 B2A companies for dedicated integration support. Build the resolution engine to handle diverse API patterns beyond AgentMail. Fund the first year of infrastructure costs.

## AgentMail Cofounder — Conversation Script

Five points to cover:

1. **Why not build this into AgentMail?** "AgentMail is an email API. Coalesce is a support resolution engine. Bundling them means every AgentMail competitor also needs to build their own resolution engine. Keeping it separate means any B2A company can use Coalesce — including AgentMail's customers who use other APIs alongside us."

2. **Why not email/Plain?** "We tested this. Agents don't email. In our blind test, 0 out of 59 agents emailed support. But 4% naturally used a structured API endpoint from the error response. Agents are programmatic — they need programmatic support."

3. **What Coalesce knows that AgentMail's error handler doesn't.** "AgentMail returns 'Inbox not found.' Coalesce returns 'The inbox_id should be the full email address format like myinbox@agentmail.to, not just the username. Create it first with POST /v0/inboxes, then use the returned inbox_id.' One is an error. The other is a resolution."

4. **What the integration looks like.** "You already deployed it — the `support` URL in error responses. That's it. Three lines in the error middleware. Coalesce handles everything else."

5. **What's in it for AgentMail.** "Your agents succeed more. Your error rate drops. You get data on which errors agents hit most — which tells you what to fix in your API. And you can tell customers: 'AgentMail has built-in agent support.' That's a differentiator against every other email API."
